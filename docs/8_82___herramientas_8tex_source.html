18 \subsubsection{Visual Studio 2019 community}
19 
20 Un entorno de desarrollo integrado \cite{visual}, en inglés Integrated Development Environment (IDE), en la versión gratuita de Microsoft y para los sistemas operativos Windows y macOS. Es compatible con múltiples lenguajes de programación, tales como C++, C sharp, Visual Basic .NET, Java, Python, Ruby y PHP. Es compatible con el framework .Net Core 5.0.
21 
22 Entre sus ventajas más relevantes podemos destacar:
23 
24 \begin{enumerate}
25 
26  \item
27  Permite escribir variables de manera rápida y precisa con las sugerencias de código de IntelliSense si el desarrollador se atasca. Mantiene la velocidad independientemente de la complejidad mientras navega a cualquier declaración de archivo, tipo, miembro o símbolo que se necesite. Realiza mejoras rápidamente en el código mediante las bombillas que sugieren acciones (por ejemplo cambiar el nombre de una función o agregar un parámetro).
28 
29  \item
30  Ayuda, mediante la característica Codelenz, a encontrar fácilmente información importante, como los cambios realizados en el código, el impacto de dichos cambios y si el método se sometió a pruebas unitarias. Permite ver referencias, autores, pruebas, el historial de confirmaciones y otros datos esenciales de un vistazo.
31 
32  \item
33  Permite pausar la ejecución de código cuando se quiere inspeccionar un error, mediante el punto de interrupción y el método necesarios. Si el desarrollador ha ido un paso demasiado lejos o detectado un cambio inesperado, puede retroceder a cualquier línea de código específica, sin tener que reiniciar la sesión o volver a crear el estado.
34 
35  \item
36  Da al desarrollador la opción de navegar por el conjunto de pruebas y de organizarse fácilmente para poder analizar la cantidad de código que prueba y ver los resultados al instante. Permite conocer, inmediatamente, el impacto de cada cambio que realiza. Para ello, hay que dar la opción de que las características avanzadas prueben el código mientras escribe. Permite , también, corregir errores a medida que se producen y ver si las pruebas existentes cubren los cambios nuevos.
37 
38 \end{enumerate}

73 \subsubsection{GitHub}
74 Desarrollado por Microsoft, es un servicio de hosting de repositorios almacenados en la nube. O sea, una plataforma donde se puede alojar código para el control de versiones y la colaboración. Permite al desarrollador y a otras personas, trabajar juntos en proyectos desde cualquier lugar. Los repositorios pueden contener carpetas y archivos, imágenes, vídeos, hojas de cálculo y conjuntos de datos, todo lo que necesite un proyecto dado.
75 
76  \subsubsection{GitHub Actions}
77  Herramienta que da la posibilidad de descubrir, crear y compartir acciones para realizar cualquier trabajo que se desee, incluido CI / CD, y combinar acciones en un flujo de trabajo completamente personalizado. Dicho de otra manera, permite al desarrollador crear su estrategia de integración continua (práctica que le requiere añadir frecuentemente nuevo código a un repositorio compartido para detectar errores a la mayor brevedad posible), dándole la posibilidad de automatizar los distintos pasos de compilación y test de su proyecto.
78 
79  \subsubsection{GitHub Desktop}
80  Con esta herramienta el desarrollador puede agregar cambios a su compromiso de forma interactiva; agregar coautores rápidamente a su compromiso; verificar las sucursales con solicitudes de extracción y ver los estados de CI; comparar imágenes cambiadas;… Está pensada, según sus creadores para alentar a su usuario y, en su caso, a otras personas que pudieran formar parte de su equipo, para que colaboren utilizando las mejores prácticas con Git y GitHub.
81 
82  \subsubsection{GitHub Pages}
83  Con esta herramienta podrás tener tu propia página web estática, configurar tu propia url, gestionar todo el código fuente desde tu propio repositorio y mantener incluso una página por proyecto sin costo alguno. El único límite que ofrece la versión gratuita es que solo se permite páginas estáticas y con el repositorio en modo "público".

90 \subsubsection{NUnit}
91 Software desarrollado por colaboradores de la .NET Foundation, NUnit es un marco de pruebas unitarias para todos los lenguajes .Net. Sirve al mismo propósito que JUnit realiza en el mundo Java. Con esta herramienta se facilita el proceso de pruebas, integrándose a la perfección con visual studio 2019, ofreciendo un conjunto de interfaces con las que se puede gestionar perfectamente las pruebas.

97 \subsubsection{BenchmarkDotNet}
98 Es una librería que ayuda a transformar métodos en puntos de referencia, realizar un seguimiento de su rendimiento y compartir experimentos de medición reproducibles. Advierte al desarrollador si hay algún problema con el diseño de su evaluación comparativa o las mediciones obtenidas. Los informes que la herramienta extrae se pueden presentar en diferentes formatos (CSV, HTML, etc.) y en ellos se resaltan todos los hechos importantes sobre el experimento realizado.

104 \subsubsection{StyleCop}
105 Herramienta desarrollada por Microsoft para análisis de código estático (es el análisis de software de computadora que se realiza sin ejecutar programas). Analiza el código fuente de C Sharp para aplicar un conjunto de reglas de estilo y coherencia. Una de sus ventajas más destacables, es su forma de instalación, ya que ofrece una extensión de visual studio 2019 que permite configurarlo de una forma bastante simple.

111 \subsubsection{Json.NET}
112 Json.NET es un popular marco JSON de alto rendimiento para .NET, que ayuda a la conversión entre el texto JSON y el objeto .NET, utilizando JsonSerializer. JsonSerializer convierte los objetos .NET en su texto equivalente JSON y viceversa asignando los nombres de las propiedades del objeto .NET a los nombres de las propiedades JSON.

117 \subsubsection{ImGUI.NET}
118 Este es un contenedor .NET para la biblioteca GUI de modo inmediato. Permite construir interfaces gráficas usando un estilo simple de modo inmediato. ImGui.NET es una biblioteca estándar de .NET y se puede utilizar en todos los tiempos de ejecución y sistemas operativos principales de .NET.
119 
120  \subsubsection{Dear Imgui}
121  Es una biblioteca de interfaz gráfica de usuario de GUI de C ++ que se utiliza principalmente en el desarrollo de juegos. Dear ImGui se centra en la simplicidad y la productividad usa lo que se llama paradigma de GUI de modo inmediato para aportar simplicidad y productividad al trabajo del desarrollador de software. Mientras que en las interfaces de modo retenido tradicionales, primero se crea un widget y a este se le agregan devoluciones de llamadas, en las GUI de modo inmediato, los widgets se crean y dibujan en cada marco.
122 
123  Está pensada diseñada para ser usada, fundamentalmente, en la creación de contenido y herramientas de depuración. Es independiente del renderizador en la forma en que debe proporcionar las herramientas para renderizar los datos, pero es muy fácil de integrar en el código, ya que tiene múltiples enlaces para diferentes bibliotecas de manejo de ventanas y eventos (como GLFW , SDL2 y GLUT) y múltiples renderizadores (como OpenGL, DirectX y Vulkan ).

129 \subsubsection{Veldrid}
130 Es una biblioteca de gráficos de bajo nivel para .NET. Se puede utilizar para crear juegos, simulaciones, herramientas y otras aplicaciones gráficas en 2D y 3D de alto rendimiento. Está diseñada para ser portátil, lo que significa que no está vinculado a ningún sistema operativo en particular o API de gráficos nativa., las aplicaciones creadas con Veldrid pueden ejecutarse en todas las plataformas de escritorio y móviles sin modificaciones usando los backends de Direct3D, Vulkan, Metal, OpenGL y OpenGL ES.

133 %% SFML.NET
134 %%%%%%%%%%%%%%
135 \subsubsection{SFML.NET}
136 Es una API multimedia sencilla, rápida, multiplataforma y orientada a objetos. Proporciona acceso a ventanas, gráficos, audio y red. Consta de varios módulos: Sistema (Clases de cadenas vectoriales y Unicode, subprocesos portátiles y funciones de temporizador); Ventana (Administración de dispositivos de entrada y ventanas , incluida la compatibilidad con joysticks, administración de contexto OpenGL); Gráficos (aceleración por hardware de gráficos 2D, incluidos sprites , polígonos y renderizado de texto); Audio (reproducción y grabación de audio especializada acelerada por hardware) y Red.

142 \subsubsection{Doxygen}
143 Es una herramienta de software para generar documentación a partir de fuentes C ++ anotadas, pero también es compatible con otros lenguajes de programación populares como C, Objective-C, C sharp, PHP, Java, Python, IDL (Corba, Microsoft y UNO / OpenOffice), Fortran, VHDL y hasta cierto punto D.
144 
145 Una de sus grandes cualidades es que el usuario tiene una variedad de formatos de documentación para elegir: HTML, PDF, RTF, etc. Dado que es fácilmente adaptable, funciona en la mayoría de sistemas Unix así como en Windows y Mac OS X.
146 Doxygen puede ser útil de tres modos:
147 
148 
149 \begin{enumerate}
150 
151  \item
152  Puede generar una documentación en línea (HTML) y/o un manual de referencia en Latex de un conjunto de ficheros fuente que estén documentados. También tiene soporte para generar salida en RTF (MS-Word), PostScript, PDF, HTML comprimido y páginas de manual UNIX. La documentación se extrae directamente de las fuentes, lo cual hace que sea mucho más fácil mantener la documentación consistente con el código fuente.
153 
154  \item
155  Es posible configurar Doxygen para que extraiga una estructura del código a partir de ficheros fuente no documentados (lo cual es muy útil cuando se tienen grandes distribuciones de código). También se pueden visualizar las relaciones entre varios elementos mediante la inclusión de gráficos de dependencia, diagramas de herencia y diagramas de colaboración, todos ellos generados automáticamente.
156 
157  \item
158  También puede utilizar Doxygen para la creación de documentación normal.
159 
160 \end{enumerate}
