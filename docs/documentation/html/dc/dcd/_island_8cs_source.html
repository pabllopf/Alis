31 Position Correction Notes
32 =========================
33 I tried the several algorithms for position correction of the 2D revolute joint.
34 I looked at these systems:
35 - simple pendulum (1m diameter sphere on massless 5m stick) with initial angular velocity of 100 rad/s.
36 - suspension bridge with 30 1m long planks of length 1m.
37 - multi-link chain with 30 1m long links.
38 
39 Here are the algorithms:
40 
41 Baumgarte - A fraction of the position error is added to the velocity error. There is no
42 separate position solver.
43 
44 Pseudo Velocities - After the velocity solver and position integration,
45 the position error, Jacobian, and effective mass are recomputed. Then
46 the velocity constraints are solved with pseudo velocities and a fraction
47 of the position error is added to the pseudo velocity error. The pseudo
48 velocities are initialized to zero and there is no warm-starting. After
49 the position solver, the pseudo velocities are added to the positions.
50 This is also called the First Order World method or the Position LCP method.
51 
52 Modified Nonlinear Gauss-Seidel (NGS) - Like Pseudo Velocities except the
53 position error is re-computed for each constraint and the positions are updated
54 after the constraint is solved. The radius vectors (aka Jacobians) are
55 re-computed too (otherwise the algorithm has horrible instability). The pseudo
56 velocity states are not needed because they are effectively zero at the beginning
57 of each iteration. Since we have the current position error, we allow the
58 iterations to terminate early if the error becomes smaller than b2_linearSlop.
59 
60 Full NGS or just NGS - Like Modified NGS except the effective mass are re-computed
61 each time a constraint is solved.
62 
63 Here are the results:
64 Baumgarte - this is the cheapest algorithm but it has some stability problems,
65 especially with the bridge. The chain links separate easily close to the root
66 and they jitter as they struggle to pull together. This is one of the most common
67 methods in the field. The big drawback is that the position correction artificially
68 affects the momentum, thus leading to instabilities and false bounce. I used a
69 bias factor of 0.2. A larger bias factor makes the bridge less stable, a smaller
70 factor makes joints and contacts more spongy.
71 
72 Pseudo Velocities - the is more stable than the Baumgarte method. The bridge is
73 stable. However, joints still separate with large angular velocities. Drag the
74 simple pendulum in a circle quickly and the joint will separate. The chain separates
75 easily and does not recover. I used a bias factor of 0.2. A larger value lead to
76 the bridge collapsing when a heavy cube drops on it.
77 
78 Modified NGS - this algorithm is better in some ways than Baumgarte and Pseudo
79 Velocities, but in other ways it is worse. The bridge and chain are much more
80 stable, but the simple pendulum goes unstable at high angular velocities.
81 
82 Full NGS - stable in all tests. The joints display good stiffness. The bridge
83 still sags, but this is better than infinite forces.
84 
85 Recommendations
86 Pseudo Velocities are not really worthwhile because the bridge and chain cannot
87 recover from joint separation. In other cases the benefit over Baumgarte is small.
88 
89 Modified NGS is not a robust method for the revolute joint due to the violent
90 instability seen in the simple pendulum. Perhaps it is viable with other constraint
91 types, especially scalar constraints where the effective mass is a scalar.
92 
93 This leaves Baumgarte and Full NGS. Baumgarte has small, but manageable instabilities
94 and is very fast. I don't think we can escape Baumgarte, especially in highly
95 demanding cases where high constraint fidelity is not needed.
96 
97 Full NGS is robust and easy on the eyes. I recommend this as an option for
98 higher fidelity simulation and certainly for suspension bridges and long chains.
99 Full NGS might be a good choice for ragdolls, especially motorized ragdolls where
100 joint separation can be problematic. The number of NGS iterations can be reduced
101 for better performance without harming robustness much.
102 
103 Each joint in a can be handled differently in the position solver. So I recommend
104 a system where the user can select the algorithm on a per joint basis. I would
105 probably default to the slower Full NGS and let the user select the faster
106 Baumgarte method in performance critical scenarios.
107 */

109 /*
110 Cache Performance
111 
112 The Box2D solvers are dominated by cache misses. Data structures are designed
113 to increase the number of cache hits. Much of misses are due to random access
114 to body data. The constraint structures are iterated over linearly, which leads
115 to few cache misses.
116 
117 The bodies are not accessed during iteration. Instead read only data, such as
118 the mass values are stored with the constraints. The mutable data are the constraint
119 impulses and the bodies velocities/positions. The impulses are held inside the
120 constraint structures. The body velocities/positions are held in compact, temporary
121 arrays to increase the number of cache hits. Linear and angular velocity are
122 stored in a single array since multiple arrays lead to multiple misses.
123 */

125 /*
126 2D Rotation
127 
128 R = [cos(theta) -sin(theta)]
129  [sin(theta) cos(theta) ]
130 
131 thetaDot = omega
132 
133 Let q1 = cos(theta), q2 = sin(theta).
134 R = [q1 -q2]
135  [q2 q1]
136 
137 q1Dot = -thetaDot * q2
138 q2Dot = thetaDot * q1
139 
140 q1_new = q1_old - dt * w * q2
141 q2_new = q2_old + dt * w * q1
142 then normalize.
143 
144 This might be faster than computing sin+cos.
145 However, we can compute sin+cos of the same angle fast.
146 */

295  public void Solve(ref Profile profile, ref TimeStep step, ref Vector2 gravity, bool allowSleep)
296  {
297  float h = step.DeltaTime;
298 
299  // Integrate velocities and apply damping. Initialize the body state.
300  for (int i = 0; i < BodyCount; ++i)
301  {
302  Body b = Bodies[i];
303 
304  Vector2 c = b.Sweep.C;
305  float a = b.Sweep.A;
306  Vector2 v = b.LinearVelocity;
307  float w = b.AngularVelocity;
308 
309  // Store positions for continuous collision.
310  b.Sweep.C0 = b.Sweep.C;
311  b.Sweep.A0 = b.Sweep.A;
312 
313  if (b.BodyType == BodyType.Dynamic)
314  {
315  // Integrate velocities.
316  v += h * b.InvMass * (b.GravityScale * b.Mass * gravity + b.Force);
317  w += h * b.InvI * b.Torque;
318 
319  // Apply damping.
320  // ODE: dv/dt + c * v = 0
321  // Solution: v(t) = v0 * exp(-c * t)
322  // Time step: v(t + dt) = v0 * exp(-c * (t + dt)) = v0 * exp(-c * t) * exp(-c * dt) = v * exp(-c * dt)
323  // v2 = exp(-c * dt) * v1
324  // Taylor expansion:
325  // v2 = (1.0f - c * dt) * v1
326  v *= MathUtils.Clamp(1.0f - h * b.LinearDamping, 0.0f, 1.0f);
327  w *= MathUtils.Clamp(1.0f - h * b.AngularDamping, 0.0f, 1.0f);
328  }
329 
330  positions[i].C = c;
331  positions[i].A = a;
332  velocities[i].V = v;
333  velocities[i].W = w;
334  }
335 
336  timer.Restart();
337 
338  // Solver data
339  SolverData solverData = new SolverData
340  {
341  Step = step,
342  Positions = positions,
343  Velocities = velocities
344  };
345 
346  //Velcro: We reduce the amount of garbage by reusing the contactsolver and only resetting the state
347  contactSolver.Reset(step, ContactCount, contacts, positions, velocities);
348  contactSolver.InitializeVelocityConstraints();
349 
350  if (step.WarmStarting)
351  {
352  contactSolver.WarmStart();
353  }
354 
355  for (int i = 0; i < jointCount; ++i)
356  {
357  if (joints[i].Enabled)
358  {
359  joints[i].InitVelocityConstraints(ref solverData);
360  }
361  }
362 
363  profile.SolveInit = timer.ElapsedTicks;
364 
365  // Solve velocity constraints.
366  timer.Restart();
367  for (int i = 0; i < step.VelocityIterations; ++i)
368  {
369  for (int j = 0; j < jointCount; ++j)
370  {
371  Joint joint = joints[j];
372 
373  if (!joint.Enabled)
374  {
375  continue;
376  }
377 
378  joint.SolveVelocityConstraints(ref solverData);
379  joint.Validate(step.InvertedDeltaTime);
380  }
381 
382  contactSolver.SolveVelocityConstraints();
383  }
384 
385  // Store impulses for warm starting.
386  contactSolver.StoreImpulses();
387  profile.SolveVelocity = timer.ElapsedTicks;
388 
389  // Integrate positions
390  for (int i = 0; i < BodyCount; ++i)
391  {
392  Vector2 c = positions[i].C;
393  float a = positions[i].A;
394  Vector2 v = velocities[i].V;
395  float w = velocities[i].W;
396 
397  // Check for large velocities
398  Vector2 translation = h * v;
399  if (Vector2.Dot(translation, translation) > Settings.MaxTranslation * Settings.MaxTranslation)
400  {
401  float ratio = Settings.MaxTranslation / translation.Length();
402  v *= ratio;
403  }
404 
405  float rotation = h * w;
406  if (rotation * rotation > Settings.MaxRotation * Settings.MaxRotation)
407  {
408  float ratio = Settings.MaxRotation / Math.Abs(rotation);
409  w *= ratio;
410  }
411 
412  // Integrate
413  c += h * v;
414  a += h * w;
415 
416  positions[i].C = c;
417  positions[i].A = a;
418  velocities[i].V = v;
419  velocities[i].W = w;
420  }
421 
422  // Solve position constraints
423  timer.Restart();
424  bool positionSolved = false;
425  for (int i = 0; i < step.PositionIterations; ++i)
426  {
427  bool contactsOkay = contactSolver.SolvePositionConstraints();
428 
429  bool jointsOkay = true;
430  for (int j = 0; j < jointCount; ++j)
431  {
432  Joint joint = joints[j];
433 
434  //Velcro: We support disabling joints
435  if (!joint.Enabled)
436  {
437  continue;
438  }
439 
440  bool jointOkay = joint.SolvePositionConstraints(ref solverData);
441  jointsOkay = jointsOkay && jointOkay;
442  }
443 
444  if (contactsOkay && jointsOkay)
445  {
446  // Exit early if the position errors are small.
447  positionSolved = true;
448  break;
449  }
450  }
451 
452  // Copy state buffers back to the bodies
453  for (int i = 0; i < BodyCount; ++i)
454  {
455  Body body = Bodies[i];
456  body.Sweep.C = positions[i].C;
457  body.Sweep.A = positions[i].A;
458  body.LinearVelocity = velocities[i].V;
459  body.AngularVelocity = velocities[i].W;
460  body.SynchronizeTransform();
461  }
462 
463  profile.SolvePosition = timer.ElapsedTicks;
464 
465  Report(contactSolver.VelocityConstraints);
466 
467  if (allowSleep)
468  {
469  float minSleepTime = MathConstants.MaxFloat;
470 
471  for (int i = 0; i < BodyCount; ++i)
472  {
473  Body b = Bodies[i];
474 
475  if (b.BodyType == BodyType.Static)
476  {
477  continue;
478  }
479 
480  if (!b.SleepingAllowed || b.AngularVelocity * b.AngularVelocity > angTolSqr ||
481  Vector2.Dot(b.LinearVelocity, b.LinearVelocity) > linTolSqr)
482  {
483  b.SleepTime = 0.0f;
484  minSleepTime = 0.0f;
485  }
486  else
487  {
488  b.SleepTime += h;
489  minSleepTime = Math.Min(minSleepTime, b.SleepTime);
490  }
491  }
492 
493  if (minSleepTime >= Settings.TimeToSleep && positionSolved)
494  {
495  for (int i = 0; i < BodyCount; ++i)
496  {
497  Body b = Bodies[i];
498  b.Awake = false;
499  }
500  }
501  }
502  }

510  internal void SolveToi(ref TimeStep subStep, int toiIndexA, int toiIndexB)
511  {
512  Debug.Assert(toiIndexA < BodyCount);
513  Debug.Assert(toiIndexB < BodyCount);
514 
515  // Initialize the body state.
516  for (int i = 0; i < BodyCount; ++i)
517  {
518  Body b = Bodies[i];
519  positions[i].C = b.Sweep.C;
520  positions[i].A = b.Sweep.A;
521  velocities[i].V = b.LinearVelocity;
522  velocities[i].W = b.AngularVelocity;
523  }
524 
525  //Velcro: We reset the contact solver instead of craeting a new one to reduce garbage
526  contactSolver.Reset(subStep, ContactCount, contacts, positions, velocities);
527 
528  // Solve position constraints.
529  for (int i = 0; i < subStep.PositionIterations; ++i)
530  {
531  bool contactsOkay = contactSolver.SolveToiPositionConstraints(toiIndexA, toiIndexB);
532  if (contactsOkay)
533  {
534  break;
535  }
536  }
537 
538  // Leap of faith to new safe state.
539  Bodies[toiIndexA].Sweep.C0 = positions[toiIndexA].C;
540  Bodies[toiIndexA].Sweep.A0 = positions[toiIndexA].A;
541  Bodies[toiIndexB].Sweep.C0 = positions[toiIndexB].C;
542  Bodies[toiIndexB].Sweep.A0 = positions[toiIndexB].A;
543 
544  // No warm starting is needed for TOI events because warm
545  // starting impulses were applied in the discrete solver.
546  contactSolver.InitializeVelocityConstraints();
547 
548  // Solve velocity constraints.
549  for (int i = 0; i < subStep.VelocityIterations; ++i)
550  {
551  contactSolver.SolveVelocityConstraints();
552  }
553 
554  // Don't store the TOI contact forces for warm starting
555  // because they can be quite large.
556 
557  float h = subStep.DeltaTime;
558 
559  // Integrate positions.
560  for (int i = 0; i < BodyCount; ++i)
561  {
562  Vector2 c = positions[i].C;
563  float a = positions[i].A;
564  Vector2 v = velocities[i].V;
565  float w = velocities[i].W;
566 
567  // Check for large velocities
568  Vector2 translation = h * v;
569  if (Vector2.Dot(translation, translation) > Settings.MaxTranslation * Settings.MaxTranslation)
570  {
571  float ratio = Settings.MaxTranslation / translation.Length();
572  v *= ratio;
573  }
574 
575  float rotation = h * w;
576  if (rotation * rotation > Settings.MaxRotation * Settings.MaxRotation)
577  {
578  float ratio = Settings.MaxRotation / Math.Abs(rotation);
579  w *= ratio;
580  }
581 
582  // Integrate
583  c += h * v;
584  a += h * w;
585 
586  positions[i].C = c;
587  positions[i].A = a;
588  velocities[i].V = v;
589  velocities[i].W = w;
590 
591  // Sync bodies
592  Body body = Bodies[i];
593  body.Sweep.C = c;
594  body.Sweep.A = a;
595  body.LinearVelocity = v;
596  body.AngularVelocity = w;
597  body.SynchronizeTransform();
598  }
599 
600  Report(contactSolver.VelocityConstraints);
601  }

638  private void Report(ContactVelocityConstraint[] constraints)
639  {
640  if (contactManager == null)
641  {
642  return;
643  }
644 
645  for (int i = 0; i < ContactCount; ++i)
646  {
647  Contact c = contacts[i];
648 
649  //Velcro feature: added after collision
650  c.FixtureA.AfterCollision?.Invoke(c.FixtureA, c.FixtureB, c, constraints[i]);
651  c.FixtureB.AfterCollision?.Invoke(c.FixtureB, c.FixtureA, c, constraints[i]);
652 
653  //Velcro optimization: We don't store the impulses and send it to the delegate. We just send the whole contact.
654  contactManager.PostSolve?.Invoke(c, constraints[i]);
655  }
656  }
