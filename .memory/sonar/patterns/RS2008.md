# Pattern: RS2008 — Enable analyzer release tracking

## Issue

RS2008 fires when an analyzer project declares `DiagnosticDescriptor` rules with IDs not found in the `AnalyzerReleases.Unshipped.md` file.

## Fix

Create `AnalyzerReleases.Unshipped.md` at the analyzer project root with:
```markdown
## Unshipped

| Rule ID | Category | Severity | Notes |
|---------|----------|----------|-------|
| ALIS00X | AOT/... | Error | Description |
```

## Requirements

- Project must have `<EnforceExtendedAnalyzerRules>true</EnforceExtendedAnalyzerRules>`
- Project must have `<EnableAnalyzerReleaseTracking>true</EnableAnalyzerReleaseTracking>`
- File must start with `## Unshipped` on line 1
- All declared `DiagnosticDescriptor` IDs must have a matching row in the table

- `6_Ideation/Fluent/generator/AnalyzerReleases.Unshipped.md`
